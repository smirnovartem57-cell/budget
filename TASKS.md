## Новые фичи (F*)

| ID | Задача | Статус | Комментарий |
|----|--------|--------|-------------|
| F1 | Свайпы на строках списков | ✅ | Свайп по карточке-строке (pointer events, порог ~28% ширины) открывает меню действий этой строки — в нём удаление и контекстные действия. Вертикальный скролл не блокируется (жест захватывается только при доминировании гориз. смещения); на десктопе отключён; кнопки («…», карандаш) сохранены. Осознанно открываем меню, а не удаляем мгновенно — защита от случайного свайпа в одностраничном приложении без общей отмены по строкам. |
| F2 | Bottom sheet быстрого добавления расхода | ✅ | Переключатель «Быстро/Все поля». В быстром режиме: крупное поле суммы (30px, по центру), чипы «частых категорий» по фактической частоте использования (kind-aware, добор из существующих категорий), добавление одним касанием без обязательного «Названия». В полном — прежний набор полей. |
| F3 | Скан чека камерой | ⏸️ | Отложено пользователем (2026-07-17). Требует серверного AI-ключа как секрета Supabase Edge Function. Клиент (камера/сжатие/экран подтверждения) + функция готовы к внедрению, когда ключ будет задан. |
| F4 | Голосовой ввод в F2 | 🟡 | Голос добавлен в bottom sheet рядом с полем суммы (разбор: сумма + категория + остаток в комментарий, общий с полем комментария). Общий экран подтверждения вместе с F3 (скан чека) — ещё нет. |
| F5 | Push-напоминания (PWA) | ⛔ | Web Push + VAPID + Edge Function + cron + таблица подписок. **Отдельная сессия** (серверная часть). |
| F6 | App Shortcuts в manifest | ✅ | В manifest.json добавлены shortcuts «Добавить расход» (`?action=expense`) и «Добавить доход» (`?action=income`). На старте `handleUrlAction` открывает лист «Новая операция» в быстром режиме с нужным типом и чистит URL. |
| F7 | Face ID поверх PIN | ✅ | WebAuthn (платформенный аутентификатор): тумблер «Включить вход по Face ID/отпечатку» в настройках PIN появляется только при `isUVPAA()` и заданном PIN; на экране блокировки — кнопка «Войти по Face ID», при любой ошибке откат на ввод PIN. Ключ не покидает устройство. Гейтинг проверен в харнессе (в file:// аутентификатора нет → тумблер скрыт, ошибок нет); сам биометрический запрос работает только на реальном устройстве с Face ID/Touch ID. |
| F8 | Расшифровка сумм по тапу на KPI | ✅ | Тап по KPI (Остаток hero + Доход/Расход/Долги) открывает единый лист `openKpiBreakdown`: период, итог, формула, что учтено, топ-3 статьи/источника/люди, время обновления. Данные считаются кодом. |
