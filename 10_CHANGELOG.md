- §6: на ширине **от 1100px** главная раскладывается в 3 колонки (одна семантическая разметка,
  CSS Grid, без дублирования HTML):
  - **Левая (≈300px):** остаток за месяц (крупная карта), Доход/Расход, Долги;
  - **Центральная (≈560px, главная):** «Новая операция» + список «Операции»;
  - **Правая (≈300px):** «Предстоит в этом месяце», «Последние операции», тизер аналитики,
    «Список покупок», быстрые действия.

  - `<1100px` — прежняя одна колонка (обёртки `display:contents`, порядок блоков = мобильный,
    ничего не переставлено). Проверено на 390/768/1024/1280/1440 — без горизонтального скролла,
    длинные названия обрезаются, ошибок в консоли нет.
  - Примечание: сущности «счета и карты» в проекте нет (данные — операции/долги/подписки), поэтому
    в левую колонку помещены реальные блоки: остаток + Доход/Расход + Долги.

## 2026-07-20 — Face ID: убрали лишний лист «Использовать ключ входа»

- В запросе биометрии добавлен `transports: ['internal']` — подсказка, что ключ на этом же
  устройстве. iOS перестаёт показывать промежуточный лист выбора ключа и уходит сразу в Face ID.
- Регистрация ключа теперь `residentKey: 'discouraged'` — не создаём discoverable-пасскей,
  чтобы не всплывал лист выбора. Для полного эффекта на уже настроенном устройстве нужно один
  раз переключить Face ID в «PIN и Face ID» (выключить → включить), чтобы ключ пересоздался.

- Для ежемесячных доходов правка касается только текущего месяца (строки
  материализованы помесячно, `amount` не трогается → следующий месяц ждёт исходную).

### Правила определения дубля (детерминированно, в копейках, без float)
- Отдельная тестируемая функция `findIncomeCandidates` (+ `moneyEqual`,
  `moneyWithinPct`, `daysApart`).
- Точное совпадение (категория + сумма до копейки + дата ±5 дней) и ровно один
  кандидат → автосвязь без создания операции, тост «Связано с доходом от …» +
  «Отменить» (5 сек).
- Похожее (±3%) либо несколько кандидатов → диалог «Похоже, этот доход уже внесён»
  со списком кандидатов + «Это новый доход».
- Отличие ≥ порога / нет кандидатов → создаётся новая операция.
- Уже связанные операции (`linked_transaction_id`) в поиске кандидатов исключаются;
  поиск ведётся по введённой сумме.
