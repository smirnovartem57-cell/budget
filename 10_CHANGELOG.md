# Changelog

Формат: обратный хронологический порядок. Основной журнал изменений — история
git-коммитов; этот файл ведёт укрупнённые записи по крупным ТЗ (по правилам проекта).

## 2026-07-17 — F1: свайпы на карточках-строках списков

- На телефоне свайп по карточке-строке любого списка (Операции, Долги, Покупки,
  Подписки, Ожидаемые доходы) открывает меню действий этой строки — с удалением и
  контекстными действиями. Реализовано на pointer-events с порогом ~28% ширины.
- Жест «захватывается» только когда горизонтальное смещение доминирует над
  вертикальным, поэтому обычный вертикальный скролл не ломается. На десктопе свайп
  отключён. Существующие кнопки («…», карандаш) остаются на месте.
- Сознательное решение: свайп открывает меню (где есть «Удалить»), а не удаляет
  мгновенно — чтобы случайный свайп не терял данные.

## 2026-07-17 — F2: быстрый bottom-sheet добавления + F4: голос в шите

### F2 — «Быстро / Все поля»
- В листе «Новая операция» появился переключатель режимов «Быстро / Все поля».
- Быстрый режим: крупное поле суммы (по центру), ряд чипов «частых категорий»,
  добавление одним касанием. «Название» в быстром режиме не требуется.
- Чипы строятся по фактической частоте использования категорий (kind-aware,
  пересобираются при переключении Доход/Расход), с добором из существующих категорий,
  если истории мало. Раньше чипы были статичны (Кредит/Обязательный/Покупки/Продукты).
- Полный режим — прежний набор полей (категория, название, дата/время, ежемесячный,
  комментарий).

### F4 — голос в bottom sheet
- Рядом с полем суммы — кнопка микрофона: распознаёт «300 продукты на ужин» →
  сумма 300, категория Продукты, остаток в комментарий. Общий разбор с полем
  комментария (`applyTxVoiceTranscript`). Экран подтверждения вместе со сканом
  чека (F3) — отдельным этапом.

## 2026-07-17 — ТЗ мобильного аудита: A1/A2 (демо-баннер, KPI 2×2) + A3 (hero-главная)

### A1 — демо-баннер без техинструкции
- Техническая подсказка (SUPABASE_URL / index.html / README) показывается только
  локально; на проде при отсутствии подключения — нейтральное сообщение. Демо-баннер
  дружелюбный, без «внутренностей».

### A2 — KPI 2×2 и устранение дублирования
- Вкладки Долги/Покупки/Доходы сокращены до 2 KPI (профильный + Остаток), без повтора
  Расход/Долги. Сетка 2 колонки на телефоне. Бюджет сохраняет полный hero-набор.

### A3 — hero-главная
- В hero-остаток добавлена шаблонная фраза-прогноз: «До конца месяца обязательных
  платежей на X, свободно примерно Y» (или «Обязательных платежей до конца месяца
  больше нет»). Считается детерминированно в копейках (`computeObligatoryRemaining`):
  активные ежемесячные расходы, не оплаченные в этом месяце, + подписки со списанием
  позже в этом месяце. Показывается только для текущего месяца.
- Три отдельные карточки (Доход/Расход/Долги) заменены компактным рядом из трёх метрик
  под hero (`.mini-metrics`), без декоративных мини-графиков.
- Панель «Ближайшие платежи» заменена лентой «Предстоит в этом месяце» (`renderUpcomingFeed`):
  объединяет ежемесячные расходы и подписки (списания) с ожидаемыми доходами (поступления)
  за текущий месяц, сортировка по дате. Направление различается иконкой (↙/↗), знаком (+/−)
  и цветом — не только цветом.

## 2026-07-15 — Долги: частичное погашение (2a) + Покупки: «Куплено» (3a)

### Долги — частичное погашение (B2/2a)
- Новая таблица `debt_payments` (история платежей). В меню долга — «Погашен
  полностью» и «Погашен частично» (ввод суммы через общий лист подтверждения,
  дата = сегодня).
- Остаток пересчитывается детерминированно в копейках (`debtRemaining`), исходная
  сумма долга не перезаписывается. При достижении нуля долг помечается погашенным.
- KPI «Непогашенные долги» и группировка «по людям» считаются по остатку. В строке —
  «Погашено X из Y». Отмена платежа — по тосту «Отменить».

### Покупки — «Куплено» (C1/C3/3a)
- «Куплено» открывает подтверждение суммы (предзаполнена, дата = сегодня) и создаёт
  расход категории «Покупки» с введённой суммой; позиция помечается купленной
  (`is_bought`), не удаляется. «Итого запланировано» её исключает.
- Купленные скрыты из списка, переключатель «Показать купленные (N)». «Вернуть в
  список» удаляет созданный расход. Отмена — по тосту «Отменить».

### Общее
- Лист подтверждения суммы+даты вынесен в общий компонент `openAmountSheet`,
  переиспользуется «Получен» / частичным погашением / «Куплено».

## 2026-07-15 — «Получен»: корректировка суммы + правила дублей

### Корректировка суммы при получении
- Кнопка «Получен» на ожидаемом доходе больше не создаёт операцию молча —
  открывается компактное подтверждение (сумма предзаполнена ожидаемой и выделена
  для быстрой правки, дата получения = сегодня, обе редактируемы; input ≥16px,
  тап-зоны ≥44px).
- Операция создаётся с ВВЕДЁННОЙ суммой. Ожидаемый доход хранит обе величины:
  `amount` (ожидаемая, не перезаписывается) и `received_amount` (фактическая) +
  `linked_transaction_id`. Новые колонки добавлены миграцией.
- Если сумма изменена — в списке показывается «Получено X из Y».
- Для ежемесячных доходов правка касается только текущего месяца (строки
  материализованы помесячно, `amount` не трогается → следующий месяц ждёт исходную).

### Правила определения дубля (детерминированно, в копейках, без float)
- Отдельная тестируемая функция `findIncomeCandidates` (+ `moneyEqual`,
  `moneyWithinPct`, `daysApart`).
- Точное совпадение (категория + сумма до копейки + дата ±5 дней) и ровно один
  кандидат → автосвязь без создания операции, тост «Связано с доходом от …» +
  «Отменить» (5 сек).
- Похожее (±3%) либо несколько кандидатов → диалог «Похоже, этот доход уже внесён»
  со списком кандидатов + «Это новый доход».
- Отличие ≥ порога / нет кандидатов → создаётся новая операция.
- Уже связанные операции (`linked_transaction_id`) в поиске кандидатов исключаются;
  поиск ведётся по введённой сумме.

## 2026-07-15 — Мобильная вёрстка и юзабилити вкладки «Бюджет»

### Общий компактный мобильный компонент строки
- `.responsive-card-table` на ширинах ≤767px теперь рендерит каждую строку как
  компактную карточку: главный текст + сумма справа (nowrap, tabular-nums) в первой
  строке, вторичная информация (дата · комментарий) ниже, действия свёрнуты в одну
  кнопку «…» (action sheet). Роль ячеек задаётся классами `rc-primary` /
  `rc-secondary` / `rc-amount` / `rc-actions`. Раньше каждая `<td>` разворачивалась
  в блок на всю ширину — карточки были высокими с пустой правой половиной.
- Применено ко всем спискам: Подписки, Ежемесячные расходы, Операции, Долги,
  Покупки, Ожидаемые доходы. Комментарий — в одну строку с ellipsis, полный текст
  по тапу (`.rc-expanded`).
- Даты во вторичных строках — в человекочитаемом виде «28 июля» (`formatDateHuman`).

### Подписки
- Форма добавления переверстана в 2 строки (Название + Сумма / Дата списания +
  Добавить) с видимым лейблом «Дата списания» (раньше поле даты было без подписи).
- Бейдж «X ₽/мес · N активных» — строчными буквами (`badge-normalcase`).

### Графики
- «Динамика по месяцам»: легенда перенесена над график, метки оси X сокращены до
  «авг '25», по умолчанию 6 месяцев с переключателем 6/12 (тап по бейджу),
  tooltip по тапу.
- «Факт по месяцам» и «Сравнение периодов»: для незавершённого текущего месяца
  показывается пометка «месяц не завершён»; вводящий в заблуждение % и зелёная
  плашка не показываются (`isCurrentMonthActive`).

### Нижняя навигация
- Матовое стекло (`backdrop-filter: blur(16px) saturate(180%)`, fallback #fff),
  граница `0.5px rgba(0,0,0,.08)`, неактивные иконки/подписи `#64748b`, активная —
  синяя. FAB «+» встроен в панель (вылет ~10px вместо 28px). Отступ контента
  увеличен, чтобы последний блок каждой вкладки доскролливался.

### Прочее
- Быстрый ввод «Список покупок на сегодня»: placeholder «Что купить?», кнопка
  «Добавить» → иконка «+» 44×44.
- Быстрые действия: подзаголовки скрыты на ≤390px, одинаковая высота карточек.
- Все инлайновые input в таблицах — font-size 16px во всём мобильном диапазоне
  (≤767px), чтобы iOS не зумил при фокусе.
- Bottom-sheet: `max-height: 88dvh` (учёт адресной строки iOS).
